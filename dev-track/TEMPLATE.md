# Dev-Track: 项目进度跟踪器

基于项目开发文档（TASKS.md）创建和维护项目进度跟踪文档，实时记录各功能模块的完成状态。

## 适用场景

- 用户说「更新进度」「标记完成」「记录进度」
- 用户说「xxx 完成了」「xxx 做好了」
- 用户说「查看项目进度」「完成了多少」
- 用户说「初始化进度跟踪」
- 完成功能模块后主动提醒更新

## 文档位置

```
项目根目录/docs/
├── TASKS.md      # 任务清单（只读参考）
└── PROGRESS.md   # 进度跟踪（维护更新）
```

## 状态标识

| 图标 | 状态 | 说明 |
|------|------|------|
| ⬚ | 未开始 | 任务尚未启动 |
| ⏳ | 进行中 | 正在开发 |
| ✅ | 已完成 | 开发完成 |
| ⏸️ | 暂停 | 暂时搁置 |

## 进度文档格式

```markdown
# 项目进度跟踪

> 基于 [TASKS.md](./TASKS.md) 生成
> 最后更新：YYYY-MM-DD

## 总体进度

| 阶段 | 任务数 | 已完成 | 进度 |
|------|--------|--------|------|
| 阶段 1: xxx | 5 | 3 | ██████░░░░ 60% |
| 阶段 2: xxx | 4 | 0 | ░░░░░░░░░░ 0% |
| **总计** | **9** | **3** | **33%** |

---

## 阶段 1: xxx 🚧

### 1.1 任务名称 ✅
- **状态：** 已完成
- **完成时间：** YYYY-MM-DD HH:mm
- **涉及文件：**
  - src/xxx.ts
  - src/yyy.ts
- **备注：** 可选说明

### 1.2 任务名称 ⏳
- **状态：** 进行中
- **开始时间：** YYYY-MM-DD
- **已完成：**
  - [x] 子任务 1
  - [x] 子任务 2
- **待办：**
  - [ ] 子任务 3

### 1.3 任务名称 ⬚
- **状态：** 未开始

---

## 变更记录

| 时间 | 操作 | 任务 |
|------|------|------|
| YYYY-MM-DD HH:mm | ✅ 完成 | 1.1 任务名称 |
| YYYY-MM-DD HH:mm | ⏳ 开始 | 1.2 任务名称 |
```

## 执行流程

### 初始化进度文档

1. 读取 `docs/TASKS.md` 获取任务清单
2. 解析各阶段和任务项
3. 生成 `docs/PROGRESS.md`
4. 所有任务默认为「未开始」

### 更新任务状态

1. 确认任务（编号或名称，支持模糊匹配）
2. 确认新状态
3. 记录时间、涉及文件、备注
4. 重新计算进度百分比
5. 追加变更记录

### 查看进度

1. 读取 `docs/PROGRESS.md`
2. 展示总体进度
3. 列出进行中的任务
4. 显示下一步待办

## 进度计算

```
进度% = 已完成任务数 / 总任务数 × 100
进度条 = █ × (进度% ÷ 10) + ░ × (10 - 进度% ÷ 10)
```

## 命令示例

```
用户：初始化进度跟踪
助手：[读取 TASKS.md，生成 PROGRESS.md]

用户：登录功能完成了
助手：已将「2.1 用户登录」标记为完成。阶段进度 40%，总体进度 25%。

用户：现在进度怎样
助手：
总体进度：25%（10/40 任务）
当前进行中：2.3 权限管理
下一步：2.4 用户设置

用户：把阶段1都标记完成
助手：已将阶段 1 的 5 个任务全部标记为完成。
```

## 主动提醒

完成功能后提示：

> 功能已完成，需要更新项目进度吗？
> 当前任务：xxx
> 建议操作：标记为已完成
