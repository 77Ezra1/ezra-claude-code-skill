# Pre-Dev: 开发前指南

当用户请求开发新功能或编写代码时，在开始编写任何代码之前，必须遵循以下流程。

## 适用场景

- 用户请求实现新功能
- 用户请求修改现有功能
- 用户描述了开发任务但细节不够清晰
- 任何需要编写或修改代码的请求

## 执行流程

### 步骤 1: 需求澄清

在开始任何实现之前，必须先与用户确认以下内容：

1. **功能目标**: 这个功能的核心目的是什么？
2. **使用场景**: 谁会使用这个功能？在什么情况下使用？
3. **输入输出**: 期望的输入是什么？期望的输出或结果是什么？
4. **边界情况**: 有哪些特殊情况需要处理？
5. **限制条件**: 有什么技术限制或业务规则需要遵守？

使用 `AskUserQuestion` 工具主动询问用户，确保理解清晰后再继续。

### 步骤 2: 方案确认

在需求明确后，提出实现方案供用户确认：

1. **技术方案**: 简要描述实现思路和技术选型
2. **文件变更**: 列出需要修改或新增的文件
3. **潜在风险**: 说明可能的风险或需要注意的地方
4. **替代方案**: 如果有其他可行方案，一并列出供用户选择

必须等待用户明确确认方案后，才能开始编写代码。

## 重要原则

- 不要假设用户的需求，有疑问必须先问
- 不要过度设计，只实现用户明确要求的功能
- 理解现有代码结构后再动手修改
- 保持与用户的沟通，确保方向正确
