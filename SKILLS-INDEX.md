## 优先级规则

当多个 skill 可能被触发时，按以下优先级自动选择：

| 优先级 | Skill | 说明 |
|--------|-------|------|
| 95 | summarize-folder | 目录遍历和文件总结（高优先级，因操作范围大） |
| 90 | commit | 明确的提交意图，最精确 |
| 88 | daily-hot-fetcher | 全平台热门信息获取 |
| 85 | prompt-packs | 提示词库 |
| 80 | visual-progress | 可视化进度显示 |
| 75 | github-upload | 明确的上传/推送意图 |
| 70 | dev-doc | 有明确的文档生成需求 |
| 60 | pm | 产品规划阶段，想法模糊 |
| 50 | dev-track | 进度跟踪 |
| 45 | visual-interface-generator | 进度界面生成 |
| 40 | bookmark | 收藏管理 |
| 35 | backend-checker | 后端代码分析 |
| 30 | pre-dev | 通用开发任务（兜底） |

### 冲突解决逻辑

```
选择逻辑：
1. 计算每个候选 skill 的匹配分数
2. 匹配分数 = 关键词精确匹配(100) + 模式匹配(80) + 意图匹配(60)
3. 最终得分 = 匹配分数 × (优先级 / 100)
4. 选择得分最高的 skill
```

### 特殊规则

1. **commit vs github-upload**:
   - 「提交」→ commit（仅本地提交）
   - 「推送」「上传」→ github-upload（推送到远程）

2. **pm vs pre-dev**:
   - 模糊想法「我想做...」→ pm
   - 明确任务「帮我实现...」→ pre-dev

3. **dev-doc 前置条件**:
   - 需要存在 PROJECT.md 或类似规划文档
   - 否则先引导使用 pm

## 使用指南

### 对于 Claude

1. 收到用户消息后，扫描是否包含上述触发词
2. 如有匹配，计算各候选 skill 的得分
3. 选择得分最高的 skill 自动执行
4. 如无明确匹配但语义相关，询问用户确认

### 对于用户

1. 可以使用自然语言描述需求，系统会自动匹配 skill
2. 也可以直接使用斜杠命令 `/skill-name` 精确调用
3. 如果自动匹配不准确，可以明确指定 skill
